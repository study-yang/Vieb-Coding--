# 微信一键登录 + 绑定手机号 — 部署与测试指南

下面步骤帮助你在 CloudBase 上部署并在真机测试一键登录与手机号绑定功能。

## 1. 配置云函数环境变量
1. 登录 CloudBase 控制台 → 选择对应环境。
2. 进入「函数服务」→ 找到 `getUserProfile` 函数 → 点击配置 → 环境变量。
3. 添加：
   - `APPID` = 你的小程序 AppID
   - `APPSECRET` = 你的小程序 AppSecret

> 注意：不要把 `APPSECRET` 写入代码或提交到源码仓库。建议使用控制台或 CI 的密钥管理功能来注入。

## 2. 部署云函数
- 在本地修改完成后，通过 CloudBase 控制台上传或使用 `cloudbase` CLI/管理工具部署 `cloudfunctions/` 目录下的函数。

## 3. 真机测试步骤（微信开发者工具）
1. 打开微信开发者工具，选择项目，使用真机调试（或使用具有授权的测试账号）。
2. 在「我的」页（`/pages/profile/profile`）操作：
   - 点击“微信一键登录”，在弹窗授权用户信息。
   - 点击“绑定手机号”并完成授权（会调用云端 `getUserProfile` 的 `bindPhone`）。
3. 在云端的 `users` 集合中查看对应 `_openid` 文档，确认 `phone` 字段已保存。

## 4. 测试迁移旧数据场景
1. 在一台设备上使用另一个 openid 绑定手机号并创建测试账单（collection `bills`）。
2. 在另一台设备上用同一手机号绑定（getUserProfile.bindPhone）：云端逻辑会把旧 openid 的 `bills`、`budgets`、`feedbacks` 更新到当前 openid，并删除旧用户文档。
3. 确认账单迁移成功。

## 5. 常见问题与排查
- 如果绑定时报 `服务器未配置小程序凭证`，请确认 `APPSECRET` 在 CloudBase 控制台已设置并保存，且函数已重新部署/重启。
- 如果绑定次数提示限流（每日上限），说明同一用户当天触发绑定已超出上限，等第二天再试或调整 `BIND_DAILY_LIMIT`。

## 6. 可选改进
- 将手机号绑定次数的上限与异常检测结合（如短时间内多次失败自动延长冷却时间）。
- 增加前端对失败原因的更细粒度提示。
